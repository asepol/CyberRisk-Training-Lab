TP7 – Architecture Sécurisée

Contexte

Vous devez concevoir une architecture sécurisée pour un SI complet basé sur votre stack Docker.

Objectifs

  • Segmenter le réseau
  • Choisir les solutions techniques
  • Proposer une architecture sécurisée

Ressources

  • Réseaux Docker : frontend, backend, dmz

Tâches à réaliser

  1. Analyser la segmentation réseau existante.
  2. Proposer un firewalling entre les zones.
  3. Ajouter un reverse proxy (conceptuellement).
  4. Proposer une architecture Zero Trust.

Aide

  • DMZ = services exposés (DVWA, Juice Shop).
  • Backend = données sensibles (MySQL, Elasticsearch).

Livrables

Un schéma réseau + justification.
