TP10 – Analyse et Exploitation d’une Messagerie Interne (Mailhog)

Contexte

Vous êtes analyste cybersécurité dans une entreprise fictive. Plusieurs applications internes (DVWA, Nextcloud, Wiki.js, Juice Shop) envoient des emails via un serveur SMTP interne appelé Mailhog. Un employé a signalé avoir reçu un email suspect provenant du service informatique.

Votre mission est d’enquêter sur les emails capturés par Mailhog.

Objectifs

  • Comprendre le fonctionnement d’un serveur SMTP interne
  • Analyser des emails légitimes et frauduleux
  • Lire et interpréter les en‑têtes SMTP
  • Identifier un email de phishing
  • Comprendre les risques liés à la messagerie interne

Ressources

  • Mailhog
  • Script : config-email-tp.sh (génère automatiquement les emails du TP)

Tâches à réaliser

1. Découverte de Mailhog

  1. Accédez à l’interface Mailhog : http://IP_VM:8025
  2. Explorez :
    • Inbox
    • Contenu des emails
    • Onglet Raw
    • Onglet Headers

Questions

  • Quel est le rôle de Mailhog dans l’infrastructure ?
  • Pourquoi utiliser un serveur SMTP interne ?
  • Quels ports Mailhog expose‑t‑il ?

2. Analyse des emails légitimes

Le script a généré plusieurs emails :

  • DVWA Security Alert
  • Nextcloud Password Reset
  • Wiki.js Invitation
  • Juice Shop Contact

Questions

  • Quel email provient d’une application de gestion de fichiers ?
  • Quel email correspond à une alerte de sécurité ?
  • Quel email est une invitation interne ?
  • Quel email provient d’un client externe ?
  • Quel champ dans les headers indique le serveur SMTP utilisé ?

3. Analyse d’un email frauduleux

Un email suspect a été généré :

Objet : URGENT – Mise à jour de sécurité

Expéditeur : it-support@lab.local

Questions

  • Pourquoi cet email est‑il suspect ?
  • Le lien contenu dans l’email est‑il légitime ?
  • L’expéditeur est‑il authentique ?
  • Quels indices dans les headers montrent une tentative de spoofing ?
  • Quel risque cet email représente‑t‑il pour l’entreprise ?

4. Investigation technique

Dans l’onglet Headers, identifiez :

  • Le champ From
  • Le champ Return‑Path
  • Le champ Received
  • Le champ Message‑ID

Questions

  • Le champ From correspond‑il au champ Return‑Path ?
  • Quel serveur a réellement envoyé l’email ?
  • Le message a‑t‑il transité par plusieurs serveurs ?
  • Le Message‑ID semble‑t‑il légitime ? Pourquoi ?

5. Génération d’un email malveillant (swaks)

Envoyez un email malveillant simple :

swaks --to victime@lab.local \
      --from admin@lab.local \
      --server IP_MAILHOG:1025 \
      --header "Subject: Mise à jour obligatoire" \
      --body "Veuillez installer la mise à jour : http://malware.lab.local"

Questions

  • Votre email apparaît‑il dans Mailhog ?
  • Est‑il possible d’usurper un expéditeur interne ?
  • Pourquoi est‑ce dangereux ?
  • Comment une entreprise peut‑elle se protéger contre ce type d’attaque ?

Livrables

  • Analyse complète des emails légitimes
  • Analyse détaillée de l’email frauduleux
  • Captures d’écran des headers SMTP
  • Liste des anomalies détectées
  • Mesures de sécurité proposées (SPF, DKIM, DMARC, filtrage, sensibilisation)
